Практическая работа №15: Настройка системы контроля версий
Тема: Работа с системой контроля версий Git
Цель работы:
1. Ознакомиться с системой контроля версий Git и ее основными возможностями.
2. Научиться работать с репозиториями: инициализация, добавление файлов, коммиты, работа с удаленными репозиториями.
3. Освоить управление ветками, слияние изменений и разрешение конфликтов.
4. Закрепить практику работы с командной строкой и Git.
Задание для выполнения:
Часть 1. Инициализация репозитория и первые коммиты
1. Создайте локальный репозиторий:
o Создайте новую папку для проекта и перейдите в нее.
o Инициализируйте репозиторий с помощью команды:
sh
КопироватьРедактировать
git init
2. Добавьте файл в репозиторий:
o Создайте файл README.md и напишите в нем краткое описание проекта.
o Добавьте файл в индекс:
sh
КопироватьРедактировать
git add README.md
o Сделайте первый коммит:
sh
КопироватьРедактировать
git commit -m "Добавлен README.md"
3. Просмотрите состояние репозитория:
o Используйте команду:
sh
КопироватьРедактировать
git status
o Убедитесь, что все файлы добавлены и зафиксированы.
Часть 2. Работа с удаленным репозиторием
1. Создайте удаленный репозиторий на GitHub/GitLab/Bitbucket.
2. Свяжите локальный репозиторий с удаленным:
sh
КопироватьРедактировать
git remote add origin <URL-репозитория>
3. Отправьте локальный код в удаленный репозиторий:
sh
КопироватьРедактировать
git push -u origin main
4. Проверьте, что файлы успешно загружены.
Часть 3. Работа с ветками
1. Создайте новую ветку для новой функциональности:
sh
КопироватьРедактировать
git checkout -b feature-branch
2. Добавьте новый файл и сделайте коммит:
sh
КопироватьРедактировать
echo "Новая функция" > feature.txt
git add feature.txt
git commit -m "Добавлен файл feature.txt"
3. Переключитесь обратно на основную ветку:
sh
КопироватьРедактировать
git checkout main
4. Объедините ветки:
sh
КопироватьРедактировать
git merge feature-branch
5. Удалите ветку после успешного слияния:
sh
КопироватьРедактировать
git branch -d feature-branch
Часть 4. Работа с конфликтами
1. Создайте конфликтную ситуацию:
o Внесите изменения в файл feature.txt в основной ветке (main).
o Внесите другие изменения в этом же файле в новой ветке.
o Попробуйте выполнить слияние (merge) — Git предупредит о конфликте.
2. Разрешите конфликт вручную:
o Откройте файл, увидите секции с конфликтами (<<<<<<<, =======, >>>>>>>).
o Выберите правильную версию кода, удалите лишние строки.
o Добавьте исправленный файл в индекс:
sh
КопироватьРедактировать
git add feature.txt
o Завершите слияние:
sh
КопироватьРедактировать
git commit -m "Разрешен конфликт при слиянии"
Часть 5. Откат изменений
1. Просмотрите историю коммитов:
sh
КопироватьРедактировать
git log --oneline
2. Отмените последний коммит (оставив изменения в файлах):
sh
КопироватьРедактировать
git reset --soft HEAD~1
3. Отмените изменения в файле и верните его в начальное состояние:
sh
КопироватьРедактировать
git checkout -- feature.txt
